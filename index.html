About

Bootbox was originally created in 2011 with the sole purpose of wrapping JavaScript's low level dialog methods with Bootstrap's high level modal functionality. As such it is a small library focussed on providing basic programattic dialogs with minimal fuss and overhead.

It is authored and maintained by Nick Payne (@makeusabrew).

API Methods (Read the full API docs)

The library exposes three methods designed to mimic their native JavaScript equivalents. Their exact method signatures are flexible but if all arguments are supplied they are assumed to be:

  • bootbox.alert(message, button, callback)
  • bootbox.confirm(message, cancelButton, confirmButton, callback)
  • bootbox.prompt(message, cancelButton, confirmButton, callback, defaultValue)

In all cases the only required argument is message. Sensible defaults will be used in place of any missing arguments, although you're unlikely to want to ever call confirm or prompt without a callback argument otherwise you'll have no way of determining what the user's response was. Because callback is arguably the most useful parameter, the three main methods can also be invoked like so:

  • bootbox.alert(message, callback)
  • bootbox.confirm(message, callback)
  • bootbox.prompt(message, callback)

If the last argument to any method is a function, it is assumed to be the callback invoked when the dialog is dismissed.

Irrespective of arguments, each of these three methods calls a fourth method, also publicly available to create custom dialogs:

  • bootbox.dialog(message, buttons, options)

Download Version 3.0.0

The latest stable version of bootbox is 3.0.0. You can download the minified library bootbox.min.js for use in production, or the uncompressed bootbox.js for use in development. I wouldn’t recommend hotlinking to either file as Github does not serve the correct mime type which may cause security warnings to be triggered based on your browser’s settings.

If you wish you may also see a full list of available downloads or download the latest copy of the entire repository (including build files and tests) in zip or tarball format. Realistically the only reason you should be using anything other than the 3.x.x series is if you are restricted by the version of Bootstrap you’re using—see the following section for full details.

Dependencies

All versions of Bootbox rely on Twitter’s Bootstrap framework and jQuery. The exact version of Bootstrap depends on the version of Bootbox you’re using. This has become slightly more complex than I’d like but hopefully this handy table will clear things up (if it doesn’t then please tell me!):

Bootbox series Min. Bootstrap version Max. Bootstrap version Notes
3.x.x Latest 2.2.2 - Anything > Bootstrap 2.2.2 is untested
2.x.x 2.0.0 2.0.4 As you can see, Bootstrap 2.1.x was never officially supported
1.x.x 1.3.0 1.4.0 Don’t download this version of Bootbox—it’s old

All versions of the library require at least jQuery 1.7.1, though 1.8.3 or above is recommended for the 3.x.x releases.
